# DESIGN — 행사 기획 에이전트

> 시각 토큰 + 컴포넌트 카탈로그 단일 소스. 화면별 배치는 `../plans/SCREENS.md`, 상태/제품 규칙은 `../plans/DATA_MODEL.md`·`../plans/PLAN.md`.
> 방향성 한 줄: **흰 바탕 · 헤어라인 · 여백으로 위계, 데이터는 고정폭 숫자, 힘은 한 곳(동작 타임라인)에만. 티얼은 절제.**

---

## 1. 디자인 원칙 (impeccable)

1. **카드 남발 금지.** 모든 걸 박스에 넣지 말고 흰 바탕 + 1px 헤어라인 + 여백으로 면을 나눈다. 그림자는 팝오버/모달에만.
2. **데이터는 고정폭(tabular).** 예산·퍼센트·날짜·타임스탬프는 모노 + `tabular-nums` + 우측 정렬 → 장부의 정밀함.
3. **힘은 한 곳에.** 시그니처는 **동작 타임라인(run-of-show 레일)**. 나머지는 의도적으로 조용히.
4. **티얼 절제.** Primary는 버튼·활성·강조 점에만. 큰 면적엔 쓰지 않는다.
5. **문서는 명조.** 제안서 본문/사례 전문만 Noto Serif KR → "진짜 문서"의 무게.
6. **모션 최소.** 허용된 두세 개 외엔 없음. `prefers-reduced-motion` 존중.
7. **카피는 사용자 언어.** 시스템 용어가 아니라 사람이 통제하는 것으로 명명. 빈 상태·에러는 다음 행동을 안내.

### 안티패턴 (하지 말 것)
보라/파랑 그라데이션 · 글래스모피즘 · 네온 다크 기본값 · ✨🤖 아이콘 · 전부 알약(pill) · 그림자 남발 · 그라데이션 텍스트 · "AI 생각 중…" 3점 애니메이션.

## 2. 컬러 토큰

### 2.1 기준 토큰 (CSS variables)
```css
:root{
  /* surface / neutral */
  --bg:#FFFFFF;       /* 메인/카드 (앱은 풀블리드 — 별도 캔버스 배경 없음) */
  --sidebar:#F4F8F8;  /* 사이드바 */
  --inset:#F7FAFA;    /* 들어간 영역(데이터 블록) */
  --ink:#15292B;      /* 본문 (순검정 아님, 딥 티얼-잉크) */
  --ink2:#3D5052;     /* 부제목/값 */
  --muted:#6B7E80;    /* 보조 텍스트/라벨 */
  --faint:#9AABAB;    /* 플레이스홀더/비활성 */
  --line:#E8EEEE;     /* 헤어라인 */
  --line2:#D7E2E2;    /* 강한 구분선/테두리 */

  /* primary (teal) */
  --primary:#0E7C82;      /* 버튼·링크·활성 */
  --primary-deep:#0A585C; /* hover/press, 진한 글씨 */
  --tint:#E1F0F1;         /* 선택 배경 */
  --tint-soft:#EEF6F6;    /* 더 옅은 배경 */

  /* semantic */
  --success:#2F9E5E; --warning:#E0A82E; --error:#D14343; --info:#3B82C4;

  /* type */
  --mono:"IBM Plex Mono",ui-monospace,Menlo,monospace;
  --serif:"Noto Serif KR",serif;
}
```

## 4. 스페이싱 · 레이아웃

- **8px 그리드**: `4 · 8 · 12 · 16 · 20 · 24 · 32 · 48`.
- **앱 셸**: 풀블리드(full-bleed) — 창을 꽉 채운다. 캔버스 위 떠 있는 둥근 프레임·바깥 여백 없음. 면 구분은 사이드바 배경 + 헤어라인으로.
- **라운드**: 카드/inset 11–12 · 버튼 9 · 뱃지 7 · 태그 5 · 미니요소 5–6.
- **사이드바** 202px(접으면 아이콘 레일).
- **콘텐츠 폭** 최대 ~1040–1060px. **문서(제안서) 컬럼** 680–760px(읽기 좋은 줄 길이).
- 본문 패딩 22–24px, 카드 내부 14–16px.

## 5. Elevation (그림자)

| 용도 | 값 |
|---|---|
| 활성 nav 칩 | `0 1px 2px rgba(21,41,43,.04)` |
| 팝오버 / 알림 패널 / 모달 | `0 12px 28px rgba(21,41,43,.13)` |

그 외 카드는 그림자 없이 **헤어라인**으로.

## 6. 모션

- 기본 `150–250ms`, ease-out. 화면 토글 ≤200ms 소프트 전환.
- **허용된 모션 (이게 전부):**
  1. 동작 타임라인 **진행 중 노드 맥동**(`pulse`, ~2.1s)
  2. 스트리밍 **커서 점멸**(caret)
  3. D-레일 **"오늘" 핀 맥동**
- ❌ 스프링 바운스 · 패럴랙스 · 광원 펄스. `@media (prefers-reduced-motion:reduce)`에서 1–3 모두 정지.

---

## 7. 아이콘

- **Tabler line icons**(webfont), 기본 16px(라벨 옆 14–15px, 강조 17–20px).
- 색: 의미 있을 때 `--primary`, 평상시 `--muted`.
- 고정 매핑: 전송=`send`(종이비행기), 이름수정=`pencil`, 상태=점, 출처=`quote`/`database`, 예산=도넛, 잠금=`lock`, 재기획=`refresh-alert`, 승인=`shield-check`, 마감=`clock-hour-4`, 알림=`bell`, 지도=`map-pin`, 날씨=`sun`/`cloud-rain`.
- ❌ 로봇·반짝이·3D 블롭.

---

## 8. 접근성 바닥

- 키보드 포커스 링(가시), 탭 순서 논리적.
- 대비: 연배경+진글씨 조합은 대비 확보(흰 글씨를 옅은 배경에 얹지 않음).
- `prefers-reduced-motion` 존중. 최소 폰트 11px.
- 색만으로 의미 전달 금지 → 상태는 항상 **점/아이콘 + 텍스트** 병행.

## 9. 컴포넌트 카탈로그

각 항목: **용도 · 구성(anatomy) · 상태/변형 · 규칙.**

### 9.1 상태 뱃지 `badge`
- 용도: 워크스페이스/항목 상태.
- 구성: 선행 점 + 라벨. 라운드 7px, 11px/500.
- 변형: 기획중·진행중(teal) / 완료(green) / 검토 대기(amber) / 지연(red) — §2.2.
- 규칙: 연배경+진글씨+동색 점. 아이콘 대신 점을 기본으로.

### 9.2 요소 타입 마커 `et` / 행위자 마커
- 용도: 타임라인 스텝이 어떤 Pi 요소인지(et), 또는 누가 한 일인지(actor).
- 구성: 10px/600 태그, 라운드 5px. §2.4 / §2.5.
- 규칙: 요소(MCP/Ext/Skill)와 행위자(AI/사람)를 섞지 말 것.

### 9.3 잠금/계획/신선도 태그
- 확정🔒: lock 아이콘 + "확정", §2.3.
- 계획: 중립 태그.
- 재검토/변동: 앰버 stale 태그. 산출물·날씨·문서 어디서나 동일 표기.

### 9.4 버튼 `btn`
- Primary: 단색 티얼 + 흰 글씨, 라운드 9px. **그라데이션 금지.**
- Secondary: 흰 배경 + 헤어라인 + ink2.
- Ghost: 텍스트만(muted).
- 크기: 기본 8/14 패딩, `sm` 6/11.

### 9.5 사이드바 (전역 셸)
- 구성: 브랜드 마크 + `홈` + 행사 목록(상태 점) + `＋ 새 행사` + **연결 인디케이터(4/5)** + 접기.
- 활성: 흰 칩 + 미세 그림자 + 헤어라인.
- 규칙: 조용하게. 행사 점 색 = 상태(진행중 teal / 완료 green / 지연 red).

### 9.6 상단 바 `top`
- 구성: 워크스페이스명 + 이름수정 연필 + 상태 뱃지 + (우측) 액션(작업공간/내보내기 등).
- 작업공간 4번엔 `AI Agent` floating 버튼(우상단).

### 9.7 상태 스트립 + 단계 스테퍼 `statusbar`
- 용도: "진행중"이 뭘 의미하는지. 워크스페이스 상단.
- 구성: 단계 스테퍼(`기획 → 진행중 → 정산·완료`, 현재 강조) + 요약 칩(제안서 vN · 확정 N🔒 · 집행 % · 다음 마감 D-N).
- 규칙: 산출물 위에 위치. 단계는 §4.3.

### 9.8 워크스페이스 타일 (홈)
- 구성: 상태 점 + 이름 + 상태 뱃지 + 조건 한 줄(유형·인원·예산·D-N) + 진행 미터(얇은 트랙 + 모노 %).
- 규칙: 헤어라인 카드, 그림자 없음.

### 9.9 동작 타임라인 (run-of-show) `tl` — ★시그니처
- 용도: 에이전트가 *방금 한 일*을 작업 로그/큐시트로.
- 구성: 세로 척추선 + 스텝(노드 · 라벨 · **요소 마커** · 모노 도구 태그 · 우측 타임스탬프 · 결과/출처).
- 노드 상태: `done`(채워진 점) / `active`(맥동 링) / `pending`(빈 점) / `gate`(앰버 마름모).
- 진행 중 스텝: 스트리밍 텍스트는 **본문(Pretendard) + 커서 점멸**. (명조는 진짜 문서 — 제안서 §9.13·사례 전문 — 에만. 타임라인 대화엔 안 씀.)
- 규칙: 출처는 `#번호`로 클릭 시 인용 드로어. 로봇 아바타·광원 금지.
- 구분: **미래 일정(D-레일)과 혼동 금지** — 타임라인=과거 로그(점/타임스탬프), D-레일=미래(D-마커/오늘 핀).

### 9.10 승인 게이트 `gatebox`
- 구성: 앰버 박스 + 질문 + `승인하고 진행` / `수정 요청`. 타임라인 안 gate 노드와 연결.
- 규칙: 승인 전 다음 스텝(적립/내보내기)으로 못 넘어감.

### 9.11 되묻기(ask) 칩
- 구성: 질문 + "답변 대기 중" + 선택 칩들. 타임라인/대화 안에.

### 9.12 컴포저 `composer`
- 구성: 입력 필드 + **중지(개입)** 아이콘 + **종이비행기 전송**.

### 9.13 결과 문서 (제안서) + 인라인 잠금
- 본문 Noto Serif KR. 확정값은 inline **잠금 칩**(🔒 인원 120명). 인용은 위첨자 모노 `#7`.
- 노션식 편집 블록(hover 시 헤어라인 핸들만).

### 9.14 인용 드로어 `drawer`
- 구성: 우측 패널 — 사례명·메타 + **하이브리드/벡터/BM25 점수** + 명조 발췌 + 출처(`Extension case_get · pgvector`).
- 트리거: 문서/타임라인의 `#번호`.

### 9.15 버전 이력 `vh`
- 구성: 세로 미니 타임라인 — `v1(AI) → 수정(사람) → v2(AI 반영)`. 노드 색: AI=teal / 사람=amber. 행위자 마커.
- 배너: "확정값은 그대로 반영" (§9.17 banner 변형).

### 9.16 재기획 배너 `replan`
- 구성: 앰버 박스 — 바뀐 입력 목록(요소/행위자 마커) + 영향받는 산출물(stale) + `다시 기획`/`변경 검토`/`무시` + **"확정·집행 항목은 잠겨 안 바뀜"** 명시.

### 9.17 안내 배너 `banner`
- 티얼 좌측 보더 + tint-soft 배경. 확정값 반영 안내 등.

### 9.18 예산 3상태 `bbar` + 표
- 막대: `집행 │ 확정·미집행 │ 계획 잔여`(§2.6) + 범례(금액).
- 표: 항목 · 계획 · 상태(확정🔒/계획) · 집행. 숫자 우측 정렬·tabular. 초과는 `--error`로 조용히.
- 주석: "재기획은 계획만 재배분, 확정·집행 유지."

### 9.19 업체 패널 + 미니 단계 스테퍼
- 행: 아이콘 + 이름 + 확정🔒/후보 태그 + (우측) 작은 단계 라벨 + **미니 4점 스테퍼**(`후보·견적·확정·계약`).
- 노드: done(채움)/cur(링). 확정 노드에 작은 lock.
- 규칙: 행 높이 키우지 않게 점 7px로 작게. 확정부터 잠금, 계약은 집행 반영(§4.5).

### 9.20 D-레일 (마일스톤) `hrail` + 그룹
- 가로 레일: `D-30 → … → 당일` 마커 + **"오늘" 핀(맥동)** + 지난 구간 fill.
- 아래 구간 그룹: `D-N` 태그 + 제목 + 마감일 + 상태(완료/다가옴) + 할 일(담당: 기획팀/에이전트, 지연 위험은 red 태그).
- 연동: `build_checklist`(Extension) + `time`(MCP). 행사일 변경 시 전부 재계산.

### 9.21 달력 `cal`
- 월 격자, 모노 숫자. 강조: 오늘(티얼 링) / 행사일(채운 티얼) / 마일스톤(tint) / 리허설(amber). 범례 병행.

### 9.22 예산 도넛
- 6색(§2.7), 가운데 총액(모노). 무지개·네온 금지.

### 9.23 지도 카드 `map`
- 장소 핀(티얼) + 주변 핀(주차·업체, 회색) + 힌트("장소 변경 시 동선·우천·거리 재검토" → 재기획 입력).

### 9.24 날씨 카드
- 아이콘 + 상태/기온 + 강수확률(모노). 예보 변동 시 `변동` stale 태그 + (↑10%) 표시.

### 9.25 산출물 리스트 `rows`
- 헤어라인 행 리스트: 제안서/내보내기/리스크/업체 비교. stale 항목엔 `재검토` 태그.

### 9.26 알림 센터 `notif` (홈)
- 벨 + 카운트 뱃지. 패널: 그룹(`재검토 필요`/`승인 대기`/`마감 임박`) + 항목(아이콘 색=종류 · 워크스페이스 점+이름 · 메시지 · 시각/D-N). 클릭 → 해당 워크스페이스.

### 9.27 연결 상태 패널 `conn`
- 행: 아이콘 + 이름(+요소 마커) + 부제(모델/서비스) + 상태(연결됨 green / 꺼짐 grey / 재연결 amber).

### 9.28 회의록 (출처 구분)
- 행: 행위자 아이콘(사람=중립/AI=티얼) + 제목 + **행위자 마커**(사람·기획팀 / AI Agent) + 시각 + 추출 인사이트.

### 9.29 세그먼트 토글 `seg`
- inset 배경 + 흰 활성 칩. 예: `달력 | D-레일`.

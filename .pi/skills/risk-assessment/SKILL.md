# 사전 점검 종합 (risk-assessment)

행사가 굴러가기 전에 **빠뜨리면 사고/위법/배제가 되는 것들**을 한 번에 훑는다. 두 곳에서 쓴다:
- **Planner**: 기획 단계에서 누락 사전 점검.
- **critic**: 초안 검토의 "리스크/누락" 각도.

각 차원은 **해당 없음**도 명시한다(점검했는데 무관한 것과 안 본 것을 구분). 확인 불가는 "확인 불가(자료 부족)"로 — 추측으로 통과시키지 않는다.
과거 유사 행사의 사고·실패는 **`rag_query`(pi-local-rag)로 사례 `cases/*.md`를 조회**해 반영한다("과거 ○○ 행사에서 …가 문제였음").

> **점검 진행:** 채워 넣는 구조화 체크리스트는 **`assets/checklist-template.json`**(차원→항목, status/severity/owner/note). 행정 컴플라이언스(§2)의 인허가·보험·소방 상세 기준은 필요할 때만 **`references/permits-insurance.md`**(기업·일반 행사)를 펼쳐 본다(SKILL.md 본문은 점검 축만 유지).
> **학생회 행사**(새터·MT·축제/주점·체전 등)는 위험·행정이 달라 **`references/student-events.md`**를 함께 본다 — **미성년 음주·부스 화기·이동/야간 안전·학교 시설 승인**이 핵심이고, 이들은 기본 '치명' 후보다.

## 1. 리스크 (운영)
- **날씨**(옥외): 우천·폭염·한파 → 대안(실내·천막·일정). 우천 확률은 external_signals/날씨 MCP.
- **취소·지연**: 핵심 업체(케이터링·AV·연사) 취소 시 백업. 계약서 위약 조항.
- **일정 충돌**: 장소 이중예약·인접 행사·교통 통제.

## 2. 행정 컴플라이언스
- **인허가**: 집회·옥외사용·도로점용·음향 허가(지자체별 상이).
- **보험**: 행사배상책임보험. 인원·옥외·고위험 프로그램이면 필수.
- **소방·안전**: 수용인원 한도, 비상구·소화기, 일정 규모 이상 소방 신고.
- **식품**: 케이터링 업체 영업신고·위생.

## 3. 접근성
- **이동**: 휠체어 동선·경사로·승강기, 장애인 화장실·주차.
- **정보**: 수어 통역·자막·점자/큰글씨 안내(규모·대상에 따라).
- **좌석**: 휠체어석·동반자석.

## 4. 비상 대응
- **화재/정전/응급**: 대피 경로·집결지, 비상연락망, 구급함·AED, 가까운 병원.
- **현장 책임자**와 역할 분담이 정해졌는가.

## 5. 식이 안전
- **알레르기**(견과·갑각류 등) 표기·분리 조리 — 안전 직결.
- **종교·신념**(할랄·채식·코셔) 옵션. 사전 수요 조사 → 케이터링 발주에 반영.

## 출력
- 차원별 **상태(OK / 주의 / 누락 / 확인불가)** + 발견 시 **무엇이·왜·어떻게**(심각도: 치명/권고).
- critic이 쓸 때는 각 항목을 처리 주체(`writer`/`재조사`/`사용자 판단`)로 분류해 넘긴다.
- 관련: 예산 영향은 [[budget-policy]](예비비·보험), 사후 개선은 [[satisfaction-survey]].
